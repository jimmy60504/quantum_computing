C Overview of QML Methods
This appendix provides a brief overview of the three quantum machine learning approaches referenced
in Problem 2. For a detailed theoretical comparison, see Ref. [3].
Explicit Quantum Model
An explicit quantum model encodes classical input x into a quantum state via an encoding circuit
S(x), applies a trainable parameterized quantum circuit W (θ), and measures an observable
ˆ
M :
f (x; θ) = ⟨0|S†(x) W †(θ)
ˆ
M W (θ) S(x)|0⟩. (2)
The parameters θare trained via gradient descent (using the parameter-shift rule or backpropagation
through the simulator).
Implicit Quantum Kernel Method
The implicit quantum kernel method uses a fixed encoding circuit S(x) with no trainable quantum
parameters to define a kernel:
k(xi, xj ) = ⟨0|S†(xi) S(xj )|0⟩2
. (3)
The kernel matrix Kij= k(xi, xj ) is computed using a quantum circuit and then passed to a classical
support vector machine:

Data Reuploading Circuit
A data reuploading circuit interleaves L layers of data encoding and trainable gates:
U (x; θ) =
L
l=1
W (θl) S(x). (4)
The input x is re-encoded in every layer, allowing the model to express higher-frequency Fourier
components. As shown in Ref. [1], the number of layers L directly controls the accessible frequency
spectrum. See Ref. [4] for a complete implementation example.

To analyze the frequency content of a 2D function, evaluate it on a regular grid and compute the 2D

x2 = np.linspace(0.5, 1.0, 64)

Common Data Encoding Strategies
Several strategies exist for encoding classical data x∈Rn into a quantum circuit:
• Angle encoding: Apply single-qubit rotations RY (xi) or RX (xi) to each qubit. Requires n
qubits for n features.
• Amplitude encoding: Encode the data vector into the amplitudes of a quantum state, re-
quiring only ⌈log2 n⌉qubits. More qubit-efficient but the state preparation circuit is more
complex.
• Data reuploading: Re-encode the data in each circuit layer (see Appendix C). Increases
expressiveness without adding qubits.
For Problem 3, where the CNN outputs a 256-dimensional vector, you will typically need a classical
layer (e.g., nn.Linear(256, n qubits)) to reduce the dimension before quantum encoding.

Training Efficiency
Quantum circuit simulation is significantly slower than classical neural network computation. To
manage training time:
• Start with small circuits (2–4 qubits, 1–2 layers) and scale up once your pipeline works.
• For Problem 3, subsample the training data during development (e.g., 5000–10000 samples) and
use the full dataset for your final run.
• GPU acceleration mainly benefits the CNN backbone. Most quantum simulators (e.g., default.qubit)
run on CPU. Plan your training time accordingly.
