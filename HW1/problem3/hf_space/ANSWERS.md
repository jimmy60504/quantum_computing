# Problem 3 — 作答

隨機種子：11224001。資料集：CIFAR-10（50,000 訓練 / 10,000 測試）。Backbone unfrozen，兩個模型一起端對端訓練 20 epochs，Adam optimizer，batch size 64。

---

## (a) 量子電路設計

### 資料編碼策略

CNN backbone 輸出的 256 維特徵向量先透過一個經典線性層降維：

```
Linear(256 → 4) + tanh(·) × π
```

此步驟將 256-D 特徵壓縮至 4 個數值，並縮放至角度編碼範圍 [−π, π]。接著透過 RY 旋轉進行 angle encoding：

```
RY(xᵢ) 作用於量子位元 i，i = 0, 1, 2, 3
```

### 變分量子電路

每個變分層由單量子位元的可訓練旋轉加上 CNOT 環狀糾纏組成：

```
對每層 ℓ = 1, 2：
    RY(θ_{ℓ,i}) · RZ(φ_{ℓ,i})  作用於每個量子位元 i
    CNOT(i → (i+1) mod 4)        環狀拓撲糾纏
```

量子可訓練參數：2 layers × 4 qubits × 2 旋轉 = **16 個量子參數**。

電路圖示（以 1 層為例）：

```
q0: ─[RY(x₀)]─[RY(θ)]─[RZ(φ)]─●───────X─  →  ⟨Z₀⟩
q1: ─[RY(x₁)]─[RY(θ)]─[RZ(φ)]─X─●─────── →  ⟨Z₁⟩
q2: ─[RY(x₂)]─[RY(θ)]─[RZ(φ)]───X─●───── →  ⟨Z₂⟩
q3: ─[RY(x₃)]─[RY(θ)]─[RZ(φ)]─────X─●─── →  ⟨Z₃⟩
              encoding       variational
```

### 量測與後處理

對每個量子位元量測 PauliZ 期望值 ⟨Zᵢ⟩，得到 4 維的實數向量（值域 [−1, 1]）。再透過線性層映射為 10 類別的 logits：

```
Linear(4 → 10) → 分類 logits
```

### 實驗設定

| 超參數 | 值 |
|---|---|
| 量子位元數 | 4 |
| 變分層數 | 2 |
| 糾纏拓撲 | CNOT 環狀（ring） |
| 梯度計算 | backprop（default.qubit） |
| CNN Backbone | Unfrozen（共同訓練） |
| 優化器 | Adam |
| Epochs | 20 |
| Batch size | 64 |

可訓練參數：Linear(256→4) [1,028] + 量子電路 [16] + Linear(4→10) [50] = **Head 共 1,094 個**；加上 backbone 56,320，總計 **57,414 個**。

---

## (b) 測試準確率

| 模型 | 最佳測試準確率 | 對應 Epoch |
|---|---|---|
| CNN + MLP（基準） | **76.81%** | Epoch 19–20 |
| CNN + QNN（4q×2l） | 61.07% | Epoch 19 |

兩者之間差距為 **15.74 個百分點**。

---

## (c) 比較表

| 模型 | 測試準確率 | 可訓練參數 | 訓練時間（20 ep） | 每 epoch 時間 |
|---|---|---|---|---|
| CNN + MLP | **76.81%** | 58,890 | 59 s | ≈ 3 s |
| CNN + QNN | 61.07% | 57,414 | 18,250 s（≈5.1h） | ≈ 912 s |

MLP head 為 `Linear(256→10)`（2,570 params）；QNN head 為 `Linear(256→4) + PQC(4q, 2L) + Linear(4→10)`（1,094 params）。兩個模型的總參數量相近（差 1,476），但測試準確率差距顯著。訓練速度方面，QNN 因量子電路串行模擬（每個樣本獨立執行 QNode），比 MLP 慢約 **311 倍**。

## (d) 訓練曲線

各 epoch 的 train/test accuracy 與 loss 曲線詳見互動介面 Training Curves 圖表。以下摘要關鍵 epoch：

| Epoch | MLP train | MLP test | QNN train | QNN test |
|---|---|---|---|---|
| 1 | 41.2% | 52.9% | 21.3% | 28.2% |
| 5 | 65.1% | 68.3% | 40.4% | 45.2% |
| 10 | 71.1% | 73.1% | 54.1% | 55.6% |

MLP 從 epoch 1 起就穩定學習，第 15 epoch 後趨於收斂。QNN 前 4 個 epoch 增益緩慢，epoch 5 後出現跳躍式進步，顯示前期可能受到梯度消失（barren plateau）的影響，需要更多 epoch 才能找到有效的梯度方向。

---

## (e) 討論

**量子電路的表達瓶頸**

本實驗以 4-qubit、2-layer PQC 取代單層線性分類頭。量子電路的 Hilbert space 為 2⁴ = 16 維，但經由 angle encoding 壓縮的輸入只有 4 個自由度，表達能力遠低於 MLP 直接在 256-D 上的線性映射。最終混淆矩陣顯示 QNN 對視覺上相近的類別（cat 13.5%、bird 28.2%）表現嚴重不足，而 frog（79.3%）、truck（80.4%）等特徵鮮明的類別反而接近 MLP 水準，印證了瓶頸來自低維的中間表示，而非量子電路本身的分類能力缺失。

**欠擬合而非過擬合**

QNN 的 train/test accuracy 幾乎一致（epoch 20：60.72% vs 61.07%），顯示模型完全沒有過擬合，而是在欠擬合——電路的容量不足以擬合訓練資料的複雜邊界。這與 MLP 的行為形成對比（MLP 在 backbone 完整的 256-D 特徵上做線性分類，資訊損失較少）。

**計算成本**

QNN 的訓練時間成本（311× 慢）是目前量子機器學習最核心的實用障礙之一。即使使用 GPU 加速的量子模擬器（lightning.gpu），PennyLane 的 autograd 不支援對 quantum circuit 的 vmap，每個樣本必須獨立執行，導致 batch processing 的計算優勢完全喪失。對比 MLP 在同等 epoch 數下只需幾十秒，QNN 需要 5 小時以上，若要訓練更多 epoch 以彌補收斂速度的劣勢，成本將更加高昂。

**結論**

在 CIFAR-10 這種有實際難度的影像分類任務上，4-qubit × 2-layer PQC 在相同參數量和 epoch 數的條件下，比單層線性 MLP 低 15.74 個百分點，且訓練時間長 311 倍。量子電路在 NISQ 時代對於經典影像分類沒有優勢；其貢獻受限於（1）低維 encoding 瓶頸、（2）訓練初期的 barren plateau、（3）無法利用 GPU 平行化的根本架構限制。
